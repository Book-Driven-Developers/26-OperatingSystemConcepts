# 운영체제 1장 정리 — 개관 (Introduction)

> 참고: Abraham Silberschatz, 『운영체제(Operating System Concepts)』
> 흔히 "공룡책"이라 불리는 이 책의 1장은 책 전체 내용을 압축해서 보여주는 챕터라, 여기를 잘 이해해두면 뒤에 나오는 각 장(프로세스, 메모리, 파일 시스템 등)을 훨씬 수월하게 읽을 수 있다.

## 1. 운영체제란 무엇인가

- **운영체제**(OS)는 컴퓨터 하드웨어를 관리하면서, 하드웨어·소프트웨어·사용자 사이를 이어주는 프로그램이다.
- **커널**(Kernel)은 이 운영체제의 실질적인 핵심부다. 운영체제는 커널 + 여러 커널 모듈로 구성되지만, 보통 "커널이 같으면 같은 OS"로 취급할 만큼 커널이 곧 OS의 정체성을 결정한다.
- 컴퓨터 시스템을 구성하는 4가지 요소
    1. 하드웨어 (CPU, 메모리, 입출력 장치 등)
    2. 운영체제
    3. 응용 프로그램 (브라우저, 워드프로세서 등)
    4. 사용자

### 비유: 운영체제 = 정부
정부 그 자체는 아무것도 "생산"하지 않지만, 사람들이 살기 좋은 환경을 만들어준다. 운영체제도 마찬가지로 그 자체가 유용한 일을 하는 건 아니지만, 다른 프로그램들이 유용한 일을 할 수 있는 환경을 마련해준다.

운영체제를 바라보는 두 가지 시각이 있다.

- **사용자 관점**(User View): 개인 PC처럼 한 사람이 독점 사용하는 환경에서는, OS가 자원 관리를 신경 쓰지 않아도 되게 해주고 사용하기 편하게 만들어주는 역할을 한다. 반면 메인프레임/미니컴퓨터처럼 여러 사람이 자원을 나눠 쓰는 환경에서는, OS가 자원을 공평하게 배분하는 역할을 한다.
- **시스템 관점**(System View): OS는 CPU 시간, 메모리, 파일 저장 공간, 입출력 장치 같은 자원을 나눠주는 **자원 할당자**(Resource allocator)이자, 이 자원들을 감독하는 **제어 프로그램**(Control program)이다.

## 2. 컴퓨터 시스템의 구성

현대 컴퓨터는 하나 이상의 CPU와 여러 **장치 컨트롤러**(Device controller)들이 공통 버스로 묶여 메모리를 공유하는 구조다.

### 부팅 과정
1. 전원을 켜면 ROM/EEPROM(펌웨어)에 저장된 **부트스트랩 프로그램**이 실행되어 시스템을 초기화한다.
2. 부트스트랩 프로그램이 **부트로더**를 실행한다. (멀티부팅 환경이면 여기서 어떤 OS를 실행할지 선택)
3. 부트로더가 최종적으로 OS 커널을 로드·실행한다.
4. 커널이 뜨면서 부팅 시 함께 로드되는 **시스템 프로세스/데몬**들이 백그라운드 서비스(로그 기록 등)를 시작한다. (UNIX 계열에서는 최초 프로세스가 `init`)
5. 부팅이 끝나면 시스템은 이벤트(인터럽트)를 기다리는 상태로 들어간다.

### 인터럽트(Interrupt)
- 장치 컨트롤러는 작업이 끝나면 CPU에게 **인터럽트**를 보내 이 사실을 알린다. CPU는 인터럽트를 받으면 하던 일을 멈추고, 고정된 메모리 위치인 **인터럽트 벡터**를 참조해 해당 인터럽트를 처리할 **서비스 루틴**으로 이동한다. 처리가 끝나면 원래 작업으로 복귀한다.
- 인터럽트는 하드웨어(시스템 버스를 통한 신호) 또는 소프트웨어(**시스템 콜**)에 의해 발생할 수 있다. 소프트웨어가 유발하는 인터럽트는 **트랩**(Trap)이라 부른다.
- 0으로 나누기 같은 내부 오류로 발생하는 인터럽트는 **예외**(Exception)라고 한다.
- 현대 OS는 대부분 **인터럽트 기반**(Interrupt-driven)으로 동작한다. 반대 방식인 **폴링**(Polling)은 이벤트를 주기적으로 계속 확인하는 방식이라 CPU 자원을 낭비하기 때문에, 인터럽트 방식이 훨씬 효율적이다.

## 3. 저장장치 구조

- 커널이 프로그램을 실행하려면, 보조기억장치에 있는 실행 파일을 **메인 메모리**(RAM)로 가져와야 한다. 모든 프로그램은 결국 메인 메모리에 로드되어야 CPU가 실행할 수 있다.
- RAM은 용량이 작고 비싸며, 전원이 꺼지면 데이터가 사라지는 **휘발성**(Volatile) 장치다. 이를 보완하기 위해 **보조기억장치**(Secondary storage)가 필요하다.
    - **비휘발성**: 자기테이프, 광디스크, 자기디스크(HDD), SSD
    - **휘발성**: 메인 메모리, 캐시, 레지스터
    - 일반적으로 용량이 클수록 저렴하고 속도가 느리며, 용량이 작을수록 빠르고 비싸다.
- **하이버네이션**(Hibernate): 절전 모드에 들어갈 때 메모리 내용을 통째로 보조기억장치에 덤프해두고, 복귀 시 그대로 불러와 이어서 작업할 수 있게 하는 기능.

## 4. 입출력(I/O) 구조와 DMA

- 컴퓨터에는 다양한 **입출력 장치**가 있고, 각 장치는 전용 컨트롤러가 담당한다. OS는 각 컨트롤러를 다루기 위한 **장치 드라이버**(Device driver)를 갖고 있다.
- I/O 흐름: 드라이버가 컨트롤러의 레지스터에 명령을 적재 → 컨트롤러가 장치와 로컬 버퍼 사이에서 데이터 전송 → 전송 완료 시 드라이버에게 인터럽트로 알림 → 드라이버가 결과를 OS에 반환.
- 사용자 프로그램은 **시스템 콜**을 통해서만 I/O를 요청할 수 있다(직접 하드웨어 접근 불가).
- **DMA**(Direct Memory Access): 예전에는 장치 데이터가 CPU를 거쳐서(=**PIO**, Programmed I/O) 메모리로 옮겨졌지만, 이는 CPU를 많이 소모한다. DMA는 장치와 메모리를 직접 연결해서, 전송이 끝났을 때 CPU에는 인터럽트 한 번만 발생시키는 방식이라 훨씬 효율적이다.

## 5. 컴퓨터 시스템 아키텍처

현대 컴퓨터는 기본적으로 **폰 노이만 구조**를 따른다.

- **단일 프로세서 시스템**(Single-Processor): 메인 CPU 하나 + 특수 목적 프로세서들(디스크 전용, 키보드 전용 등)로 구성되던 과거의 방식.
- **멀티프로세서 시스템**(Multiprocessor): 프로세서가 2개 이상.
    - 장점: 처리량 증가, 규모의 경제(주변장치 공유로 비용 절감), 신뢰성 증가(**우아한 성능저하 Graceful degradation**를 통한 **장애 허용 Fault tolerant**).
    - **비대칭 멀티프로세싱**: 보스 프로세서가 다른 프로세서들을 통제(관료적 구조). 보스가 죽으면 전체가 멈춤.
    - **대칭 멀티프로세싱**(SMP): 모든 프로세서가 동등하게 메모리를 공유하며 병렬 작업. 한 프로세서에 문제가 생기면 나머지가 작업을 나눠 받는다. 단점을 보완하기 때문에 현재 주류 방식.
- **멀티코어**(Multicore): 하나의 칩에 여러 코어를 집적한 형태. 칩 내부 통신이 칩 간 통신보다 빨라 효율적이고 전력 소모도 적다. 단, 코어 수가 늘어날수록 코어 간 통신 비용 때문에 성능 향상은 로그 함수 형태로 완화된다(사람이 늘어날수록 조율 비용이 커지는 것과 비슷).
- **클러스터 시스템**(Clustered System): 독립된 여러 시스템(노드)이 네트워크(주로 LAN)로 묶여 하나의 시스템처럼 동작. 저장소를 공유하며 **약결합**(Loosely coupled) 구조. 고가용성을 위해 사용되며 단일 고성능 컴퓨터보다 저렴하게 비슷한 효과를 낼 수 있다.
    - **비대칭 클러스터링**: 한 노드는 상시 대기(Hot-standby) 상태로 모니터링만 하다가, 문제가 생기면 대신 작동.
    - **대칭 클러스터링**: 모든 노드가 작업을 수행하면서 서로를 모니터링. 자원을 더 효율적으로 활용.

## 6. 운영체제 구조와 동작 방식

- **멀티프로그래밍**(Multiprogramming): 여러 프로그램을 메모리에 동시에 올려두고, 하나가 대기 상태가 되면 다른 프로그램을 실행시켜 CPU를 놀리지 않는 방식.
- **시분할(Time sharing) = 멀티태스킹**: 멀티프로그래밍을 더 발전시켜, 프로세스마다 짧은 시간을 할당해 빠르게 전환하며 실행. 사용자에게는 여러 작업이 동시에 실행되는 것처럼 보인다. 이때는 **반응 시간**(Response time)이 중요한 지표가 된다.
- 여러 작업을 동시에 메모리에 올리다 보면 자리가 부족할 수 있어, 어떤 작업을 먼저 처리할지 정하는 **작업/CPU 스케줄링**이 필요하다.
- 메모리가 부족할 때는 보조기억장치 일부를 메인 메모리처럼 쓰는 **가상 메모리**(Virtual memory)를 활용해, 실제 물리 메모리보다 큰 프로그램도 돌릴 수 있게 한다.

### 이중 모드(Dual-Mode) / 다중 모드 운영
- OS는 사용자 프로그램이 시스템을 함부로 건드리지 못하도록 **유저 모드**와 **커널 모드**를 구분한다. 하드웨어의 모드 비트가 0이면 커널 모드, 1이면 유저 모드다.
- **특권 명령**(Privileged instruction)은 커널 모드에서만 실행 가능하다. 유저 모드에서 이를 시도하면 하드웨어가 막고 OS에게 트랩을 던진다.
- 유저 모드 프로그램이 커널 기능을 쓰려면 반드시 **시스템 콜** 인터페이스를 거쳐야 한다.

### 타이머(Timer)
사용자 프로그램이 제어권을 계속 붙들고 놓지 않는 상황을 막기 위해, OS는 타이머로 일정 주기마다 인터럽트를 발생시켜 강제로 제어권을 되찾는다. (카운터를 설정하고 매 틱마다 감소시켜 0이 되면 인터럽트 발생)

## 7. 운영체제가 관리하는 주요 대상

- **프로세스 관리**: 프로그램은 디스크에 있는 수동적인 존재, **프로세스**는 메모리에 로드되어 실행 중인 능동적인 존재다. 프로세스는 실행 위치를 기록하는 **프로그램 카운터**를 가진다(싱글쓰레드는 1개, 멀티쓰레드는 여러 개). OS는 프로세스/쓰레드의 생성·삭제·스케줄링·중단·재개·동기화·통신을 담당한다.
- **메모리 관리**: 메인 메모리는 주소를 가진 거대한 바이트 배열이며, 프로그램은 절대 주소로 매핑되어 로드된다. OS는 메모리의 어느 영역이 누구에게 쓰이는지 추적하고, 필요에 따라 메모리를 할당·회수한다.
- **저장장치 관리**: OS는 물리적 저장장치의 세부 사항을 감추고, **파일**이라는 논리적 단위로 추상화한다.
    - **파일 시스템 관리**: 파일 생성·삭제·읽기·쓰기 등, OS의 역할 중 사용자에게 가장 눈에 띄는 부분.
    - **대용량 저장장치 관리**: WORM(한 번 쓰고 여러 번 읽기) 방식과 RW(읽기/쓰기 모두 가능) 방식의 매체가 있으며, NTFS·FAT 같은 파일 시스템 포맷의 차이 때문에 서로 다른 포맷 간에는 읽기는 되지만 쓰기가 안 되는 경우가 많다.
    - **캐싱**(Caching): 느린 디스크 접근을 줄이기 위해 자주 쓰일 데이터를 빠르고 작은 캐시에 미리 담아두는 기법. **지역성**(Locality) 원리에 기반한다.
        - **시간 지역성**: 한 번 접근한 데이터는 곧 다시 접근할 가능성이 높다.
        - **공간 지역성**: 방금 접근한 데이터 근처의 메모리도 곧 접근할 가능성이 높다 (예: 배열을 순서대로 순회하는 반복문).
    - **I/O 시스템**: OS는 모든 입출력 장치를 파일처럼 취급하며, 장치별 세부 정보는 해당 장치 드라이버만 알고 있다.

## 8. 보호와 보안

- 여러 사용자가 하나의 시스템을 공유하기 때문에 **보호**(Protection)와 **보안**(Security)이 중요하다. 유저 모드/커널 모드의 구분도 보호 메커니즘의 하나다.
- **권한 확대**(Privilege escalation): 사용자의 권한 단계를 나누고, 특정 조건에서 권한을 올려주는 구조.
    - **수직 권한 확대**: 임의 코드 실행 등을 통해 더 높은 권한을 얻는 경우.
    - **수평 권한 확대**: 안드로이드 루팅, iOS 탈옥처럼 동일 레벨에서 제약을 풀어내는 경우.

## 9. 커널 자료구조

커널 내부에는 리스트, 스택, 큐, 연결 리스트 같은 기본 자료구조가 널리 쓰이며, 특히 **트리**(Tree)는 로그 시간(O(log n))의 효율적인 탐색이 가능해 자주 활용된다.

## 10. 컴퓨팅 환경과 오픈소스 OS

- **모바일 컴퓨팅**: 다양한 센서를 통해 컴퓨터 접근성과 사용자 인터페이스를 크게 확장시켰다.
- 그 외에도 **분산 컴퓨팅**, **클라이언트-서버 컴퓨팅**, **P2P 컴퓨팅**, **클라우드 컴퓨팅**(AWS 등으로 대중화) 등 다양한 환경이 존재한다.
- 현대 컴퓨팅 환경을 요약하는 키워드: **휴대성**(Mobility), **가상화**(Virtualization), **멀티코어**(Multicore).
- **오픈소스 운영체제**: 리눅스처럼 누구나 소스 코드를 보고 개발에 참여하거나, 이를 기반으로 새로운 OS를 만들 수 있는 생태계가 존재한다.

### 한눈에 정리
| 키워드 | 핵심 요약 |
|---|---|
| 커널 | OS의 실질적 핵심, OS = 커널 + 커널 모듈 |
| 자원 할당자 | 시스템 관점에서 본 OS의 역할 |
| 인터럽트 | 이벤트 발생을 CPU에 알리는 신호 (하드웨어 인터럽트 / 소프트웨어=트랩) |
| DMA | CPU 부담 없이 장치↔메모리 직접 전송 |
| SMP | 다수 프로세서가 메모리를 공유하며 대등하게 동작하는 현재 주류 구조 |
| 시분할 | 짧은 시간 단위로 프로세스를 번갈아 실행 → 동시성처럼 보이게 함 |
| 이중 모드 | 유저 모드/커널 모드 구분으로 시스템 보호 |
| 지역성 | 캐싱 효율의 근거 (시간/공간 지역성) |
