(* 컴퓨터 시스템 -> 프로그램을 실행하고 사용자 문제를 더욱 쉽게 해결할 수 있게 하는 것)

## 컴퓨터 시스템의 구성

컴퓨터 시스템은 하나 이상의 CPU와 구성요소와 공유 메모리 사이의 액세스를 제공하는 공통 버스를 통해 연결된 여러 장치 컨트롤러로 구성되는데, 각 장치 컨트롤러마다 장치 드라이버가 있다. 
장치 드라이버는 일관된 인터페이스를 제공한다. 
CPU와 장치 컨트롤러는 병렬로 실행되며 메모리 사이클을 놓고 경쟁한다. 

### 인터럽트 

1. 컨트롤러가 장치에서 로컬 버퍼로 데이터를 전송
2. 데이터 전송이 완료되면 컨트롤러가 드라이버에게 작업 완료를 알림 <-- 인터럽트!
3. 드라이버는 운영체제의 다른 부분에게 제어를 넘김

운영체제와 하드웨어의 상호 작용 방식의 핵심 부분이다. 

CPU가 인터럽트되면 CPU는 하던 일을 중단하고 즉시 고정된 위치로 실행을 옮긴다. 고정된 위치는 일반적으로 인터럽트를 위한 서비스 루틴이 위치한 시작 주소를 가지고 있다. 그리고 인터럽트 서비스 루틴 실행, 이 실행이 완료되념 CPU는 인터럽트 되었던 연산을 재개한다.

* 인터럽트 핸들러, 인터럽트 테이블
  
#### 구현
CPU의 인터럽트 요청 라인에 컨트롤러가 신호를 보낸 것을 감지하면 인터럽트 번호를 읽고 이 번호를 인터럽트 벡터의 인덱스로 사용하여 해당되는 인터럽트 핸들러 루틴으로 점프한다. 인터럽트 처리기는 작업 중에 변경될 상태를 저장하고, 인터럽트 원인을 확인하고, 처리하고, 상태를 복원하고, return_from_interrupt 명령어를 통해 CPU를 인터럽트 전 실행 상태로 되돌린다. 

** 인터럽트는 시간에 민감한 처리에 빈번하게 사용되므로, 시스템 성능을 좋게 하려면 효율적인 인터럽트 처리가 필요하다.**

### 저장장치 구조

대부분의 컴퓨터는 프로그램 대부분을 메인메모리(RAM)에서 가져온다. 

모든 형태의 메모리는 바이트 배열을 제공하고, 각 바이트는 자기 주소를 가진다. 
메모리에 대한 상호작용은 load, store를 사용해서 이루어지는데...
1. load: 적재 명령, 메인 메모리로부터 CPU 내부의 레지스터로 옮기는 것
2. store: 저장 명령, 레지스터의 내용을 메인 메모리로 옮기는 것

메인 메모리는 너무 작고, 휘발성이기때문에 보완해줄 보조저장장치가 필요하다. -> HDD

### 입출력 구조, I/O structure

시스템의 안정성, 성능, 장치의 다양한 특성 떄문에 I/O 관리는 중요하다. 

- 소량의 데이터를 이동시킬때는 인터럽트 기반의 I/O를 써도 상관없지만, 
- 대량의 데이터를 이동시킬때는 DMA를 사용해서 장치 컨트롤러가 데이터를 전송하는동안 CPU는 다른 생산적인 작업을 할 수 있도록 한다.

## 운영체제의 작동

운영체제는 아무일 없을때는 실행할 이벤트를 기다리고 있는데, 이벤트는 거의 항상 인터럽트를 통해서 신호를 보낸다. 

### 다중 프로그래밍과 다중 태스킹
- 다중 프로그래밍: 여러 프로그램을 메모리에 올려두고, 하나가 I/O를 기다리는 동안 CPU가 다른 프로그램을 실행해 CPU 활용률을 높이는 기법
  - CPU를 놀리지 않는 것이 목적
- 다중 태스킹: CPU 시간을 매우 짧게 나누어 여러 프로세스에 번갈아 할당함으로써 사용자가 동시에 실행되는 것처럼 느끼게 하는 기법
  - 빠른 응답성과 사용자 경험이 목적

현대 운영체제는 다중 프로그래밍과 다중 태스킹을 함께 사용하며, 스케줄러가 CPU를 여러 프로세스에 번갈아 할당해서 높은 자원 활용과 응답성을 동시에 제공한다.

### 이중모드와 다중모드 운영

운영체제는 사용자 프로그램이 시스템을 손상시키는 것을 방지하기 위해 실행 모드를 구분한다.
이중 모드는 사용자 모드(User)와 커널 모드(Kernel)로 나뉘며, mode bit로 현재 실행 모드를 구분한다.

privileged instruction은 커널 모드에서만 실행할 수 있으며, 사용자 프로그램은 이를 직접 수행할 수 없다. 사용자 프로그램이 운영체제의 기능을 사용하려면 System Call을 통해서 사용자 모드에서 커널 모드로 전환한 후에야 사용할 수 있다. 
최신 CPU는 다중 모드를 지원하여 가상 머신이나 하이퍼바이저 같은 환경을 위해 2개 이상의 권한 수준을 제공할 수 있다.

## 자원 관리 
### 프로세스 관리 
프로세스는 실행 중인 프로그램으로, 운영체제는 여러 프로세스가 시스템 자원을 효율적으로 사용할 수 있도록 관리한다.
하나의 프로세스는 CPU를 사용하여 명령을 수행하고, 필요에 따라 I/O를 요청하거나 새로운 프로세스를 생성할 수 있다. 운영체제는 프로세스의 생성, 종료, Suspend, Resume 기능을 제공한다.

### 메모리 관리
운영체제는 실행 중인 프로그램이 사용할 메모리 공간을 할당하고 회수하며, 여러 프로세스가 메모리를 안전하게 공유할 수 있도록 관리한다. 또한 어떤 프로세스를 메모리에 적재할지와, 필요한 경우 메모리와 저장장치 간 데이터를 이동시키는 일을 담당한다.
컴퓨터 자원을 효율적으로 사용하기위해서 우리는 메모리에 여러개의 프로그램을 유지하기 위한 메모리 관리 기법이 필요하다. 

### 파일 시스템 관리
운영체제는 파일과 디렉터리를 생성·삭제하고, 접근 권한을 관리하며 데이터를 저장하고 검색할 수 있도록 파일 시스템을 제공, 파일을 실제 저장장치에 효율적으로 배치하고 관리하는 역할을 수행한다.

### 대용량 저장장치 관리
대부분의 데이터는 디스크와 같은 대용량 저장장치에 저장되므로, 운영체제는 저장 공간을 효율적으로 관리해야 한다. 이를 위해 저장 공간 할당, 디스크 스케줄링, 빈 공간 관리 등의 기능을 제공한다.

### 캐시 관리
캐시는 자주 사용하는 데이터를 빠르게 접근할 수 있도록 저장하는 고속 저장 공간이다.
운영체제는 여러 계층의 캐시를 효율적으로 활용하여 시스템 성능을 향상시키고, 데이터의 일관성을 유지하도록 관리한다.

### 입출력 관리
운영체제는 다양한 입출력 장치를 일관된 방식으로 사용할 수 있도록 장치 드라이버와 I/O 시스템을 제공한다. 또한 입출력 요청을 관리하고, 버퍼링·캐싱·스풀링 등을 통해 입출력 성능을 향상시킨다.


## 보안과 보호

1. 보호(Protection): 시스템 내부의 프로세스와 사용자들이 자원에 올바르게 접근하도록 제어하는 기법 
   - 누가 무엇을 사용할 수 있는지?
2. 보안(Security): 외부 및 내부의 공격으로부터 시스템을 보호하고, 무단 접근을 방지
   - 시스템을 외부로부터 어떻게 지킬 것인지?

운영체제는 사용자 인증(Authentication)과 권한 부여(Authorization)를 통해 자원 접근을 통제한다.
시스템은 최소 권한의 원칙(Principle of Least Privilege)을 적용하여 필요한 권한만 부여해야 한다.

## 가상화 
가상화(Virtualization): 하나의 물리 컴퓨터를 여러 개의 독립적인 실행 환경으로 추상화하여, 각각이 자신만의 컴퓨터를 사용하는 것처럼 동작하게 만드는 기술.

각 가상 머신(Virtual Machine)은 독립적인 운영체제와 응용 프로그램을 실행할 수 있으며, 서로 영향을 주지 않는다. 이를 가능하게 하는 소프트웨어를 가상 머신 관리자(VMM, Virtual Machine Monitor)라고 하며, VMM은 하드웨어 자원을 관리하고 각 가상 머신을 격리한다.
가상화는 동일한 CPU 아키텍처에서 운영체제를 실행하는 기술인 반면, 에뮬레이션(Emulation)은 다른 CPU 아키텍처의 명령어를 소프트웨어로 변환하여 실행하는 기술이다.
가상화는 개발·테스트 환경 구축, 여러 운영체제의 동시 실행, 데이터센터의 서버 통합 등 다양한 분야에서 널리 활용된다.


## 분산 시스템
분산 시스템: 네트워크로 연결된 여러 대의 독립적인 컴퓨터가 자원과 데이터를 공유하며 함께 동작하는 시스템

분산 시스템은 계산 성능 향상, 자원 공유, 데이터 가용성 및 시스템 신뢰성 향상을 목적으로 한다.
운영체제는 네트워크를 통해 다른 컴퓨터의 자원에 접근할 수 있도록 다양한 네트워크 프로토콜(대표적으로 TCP/IP)을 지원한다. 네트워크는 연결 범위에 따라 LAN, MAN, WAN, PAN 등으로 구분되며, 운영체제는 장치 드라이버와 네트워크 소프트웨어를 통해 이를 관리한다.

분산 시스템은 네트워크 운영체제와 분산 운영체제가 있고, 분산 운영체제는 여러 컴퓨터를 하나의 시스템처럼 보이도록 관리하며, 네트워크 운영체제는 네트워크의 다른 모든 컴퓨터가 자율적으로 작동하게 한다.

## 커널 자료구조

### 리스트, 스택, 큐

* 리스트: 데이터의 삽입과 삭제가 쉽고 크기가 유동적인 자료구조이지만, 특정 원소를 찾을 때는 순차 탐색이 필요하여 O(n)의 시간이 걸린다.
* 스택은 LIFO(후입선출) 구조로 함수 호출 시 지역 변수, 매개변수, 복귀 주소 등을 저장하는 Call Stack에 사용된다.
* 큐는 FIFO(선입선출) 구조로 CPU 스케줄링의 Ready Queue, 프린터 작업 대기열 등 작업을 순서대로 처리하는 데 사용된다.

### 트리 

* 트리는 부모-자식 관계로 데이터를 저장하는 계층적 자료구조이며, 이진 탐색 트리(BST)는 왼쪽 자식 < 부모 < 오른쪽 자식의 규칙을 따른다.
* 일반적인 BST는 최악의 경우 O(n)이지만, 균형 이진 탐색 트리는 높이를 유지하여 O(log n)의 탐색 성능을 보장한다.
* 운영체제에서는 Red-Black Tree와 같은 균형 트리를 CPU 스케줄링 등에 활용한다.


### 해시 함수와 맵

* 해시 함수는 데이터를 해시값으로 변환하여 배열의 인덱스로 사용함으로써 평균적으로 O(1)에 가까운 빠른 탐색을 가능하게 한다.
* 서로 다른 키가 같은 해시값을 가지는 해시 충돌이 발생할 수 있으며, 일반적으로 연결 리스트 등을 이용해 해결한다.
* 해시 맵은 키와 값을 저장하는 자료구조로, 운영체제에서 다양한 정보를 빠르게 관리하는 데 사용된다.

### 비트맵

* 비트맵은 1개의 비트(0 또는 1)로 하나의 자원의 상태를 표현하는 자료구조로, 매우 적은 메모리로 많은 자원의 사용 여부를 관리할 수 있다.
* 일반적으로 0은 사용 가능, 1은 사용 중을 의미하며, i번째 비트는 i번째 자원과 대응된다.
* 예를 들어 `001011101`이라면 각 비트가 하나의 자원을 나타내며, 비트값을 확인하는 것만으로 특정 자원이 사용 가능한지 즉시 알 수 있다.
* 운영체제는 디스크 블록, 메모리 페이지, inode, PID 등 대량의 자원 할당 여부를 관리할 때 비트맵을 사용한다.
* 비트맵은 자원 하나당 1비트만 사용하므로 매우 공간 효율적이며, 수천~수백만 개의 자원 상태를 관리하기에 적합한 자료구조이다.


## 계산 환경

### 전통적 계산 

* 전통적인 컴퓨팅은 개인용 컴퓨터와 서버를 중심으로 발전했으며, 오늘날에는 웹 기술과 고속 네트워크의 발전으로 활용 범위가 크게 확장되었다.
* 과거의 시분할 시스템은 여러 사용자가 하나의 컴퓨터를 공유했지만, 현대에는 하나의 사용자가 여러 프로세스를 동시에 실행하는 형태로 발전하였다.

### 모바일 컴퓨팅

* 모바일 컴퓨팅은 스마트폰과 태블릿을 이용한 컴퓨팅 환경으로, 무선 네트워크나 이동통신망을 통해 다양한 온라인 서비스를 이용한다.
* 대표적인 모바일 운영체제로는 Apple iOS와 Google Android가 있다.

### 클라이언트-서버 컴퓨팅

* 클라이언트가 요청을 보내면 서버가 이를 처리하여 결과를 반환하는 구조이다.
* 서버는 계산 서비스를 제공하는 계산 서버와 파일을 제공하는 파일 서버 등으로 구분된다.

### 피어 간 계산

* 모든 노드가 클라이언트와 서버 역할을 모두 수행하며, 중앙 서버 없이 자원을 공유하는 구조이다.
* 서버 병목 현상을 줄일 수 있지만, 피어를 찾기 위한 발견 프로토콜이 필요하다.

### 클라우드 컴퓨팅

* 클라우드 컴퓨팅은 서버, 저장소, 플랫폼, 소프트웨어 등을 인터넷을 통해 서비스 형태로 제공하는 컴퓨팅 방식이다.
* 가상화를 기반으로 하며, Public, Private, Hybrid 클라우드로 구분된다.
* 서비스 형태는 SaaS(소프트웨어), PaaS(플랫폼), IaaS(인프라)로 나뉜다.

### 실시간 내장형 시스템

* 내장형 시스템은 자동차, 가전제품, 공장 설비 등 특정 목적을 수행하는 컴퓨터 시스템이다.
* 대부분 실시간 운영체제를 사용하며, 정해진 시간 안에 작업을 완료해야 하는 시간 제약을 만족하는 것이 중요하다.
